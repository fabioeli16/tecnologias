--------------------------------------------------------------
-RENDERIZADO

*En React, renderizar NO significa “pintar el HTML en pantalla”. (Renderizado ≠ DOM)
*renderizar es Ejecutar un componente para calcular qué UI debería verse
*Un componente se renderiza cuando ocurre alguno de estos eventos:
  ✔ Se monta por primera vez
  ✔ Cambia su state
  ✔ Cambian sus props
  ✔ Cambia el context
  ✔ Su padre se renderiza (por defecto)

----------------------------------------------------------------------------------------------------------------------------------------------------------------------
ESTADOS DE UN COMPONENTE

En React, los estados de un componente (state) son datos internos que pertenecen al componente y que pueden cambiar con el tiempo.

Cuando el estado cambia, React vuelve a renderizar el componente para reflejar esos cambios en la interfaz.


--------------------------------------------------------------
-EFECTOS

En React, los efectos (effects) sirven para manejar efectos secundarios dentro de un componente.
Es decir: todo aquello que no es solo pintar HTML.

Un efecto secundario es cualquier acción que:
  *Interactúa con algo externo al render
  *O necesita ejecutarse después de que React renderiza

Ejemplos típicos:
  *Llamadas a APIs (fetch / axios)
  *Suscribirse a eventos (scroll, resize, WebSocket)
  *Manipular el DOM
  *Timers (setTimeout, setInterval)
  *Leer o escribir en localStorage
  *Logs, métricas, analytics
